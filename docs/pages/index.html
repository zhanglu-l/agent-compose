快速开始

agent-compose

agent-compose 是一个面向 agent 工作流的控制面：用本地 compose 定义 project、agent 和调度规则，再由常驻 daemon 统一负责执行、状态持久化、日志、sandbox 和运行过程管理。

快速开始 查看仓库
快速开始
agent-compose daemon

agent-compose up
agent-compose run reviewer --prompt "review this change"
先启动 daemon，再用 up 应用本地 compose，然后执行一次 agent。

agent-compose 解决什么

把零散脚本收成一个运行面

把本地 compose、调度规则、runtime 和运行状态放到同一套控制面里，不再靠分散脚本和临时命令串起来。

把 agent 执行变成可追踪流程

同一套入口下统一管理执行、日志、sandbox 和历史状态，让团队能稳定复现、排查和持续运行 agent 工作流。

把本地定义接到长期运行的控制服务

CLI 负责读取和规范化本地 compose，daemon 负责长期状态和运行期能力，两者配合形成稳定的 up、run、logs 主路径。

How It Works

工作原理

Compose Source agent-compose.yml

定义 project、workspace、variables、agents，以及 scheduler triggers 或 scheduler script。

project agents scheduler
CLI Surface CLI

负责本地解析、规范化、环境变量插值，然后调用 daemon 的 project / run / exec / image 主路径。

up run logs

daemon

状态管理、scheduler、runtime 生命周期、Connect API、workspace 路由和 Jupyter proxy

主命令路径 最常见的使用路径就是 up、run 和 logs。
兼容与边界 保留现有接口以兼容当前 Web/UI；默认本地优先，前端独立部署，不再打进 daemon 静态资源。
Execution Runtimes

通过 `boxlite`、`docker`、`microsandbox` 拉起 guest，准备 workspace，并维持 sandbox 生命周期。

boxlite docker microsandbox
Access Web/UI and Sandboxes

当前 Web/UI 继续使用现有兼容接口；sandbox、logs、workspace、proxy 路由和运行历史都由 daemon 统一协调。

existing API sandbox workspace

快速开始

1

启动 daemon

先启动长期运行的 daemon。CLI、Web/UI 和自动化调度都以它为状态来源。

agent-compose daemon
2

准备 compose 文件

定义最小 project，声明 agent、image 和 provider。

name: demo agents: reviewer: provider: codex model: gpt-test image: guest:v1
3

应用并运行

先用 `up` 把 compose 应用到 daemon，再执行一次 `run` 并查看日志。

agent-compose up agent-compose run reviewer --prompt "review this change" agent-compose logs --agent reviewer agent-compose ps

使用场景

定时代码审查与状态检查

用 hourly/daily trigger 让 reviewer agent 周期性检查仓库、工作区或运行状态。

Webhook 与共享 playground

Git push、工单事件或业务 topic 可以直接触发 loader 和 agent 任务，也适合配合独立前端做多人共享的 playground。
